제품 요구사항 정의서 (PRD) v2
1. 프로젝트 개요 (Project Overview)
프로젝트명: 개인 학습용 웹앱 큐레이션 포털 (Learning WebApp Portal)

목적: 개발자가 직접 제작한 다양한 학습용 웹앱을 한곳에 모아 체계적으로 관리하고, 사용자가 직관적으로 탐색 및 실행할 수 있는 통합 웹 포털 구축

주요 대상: 포털 제작자(관리자) 및 학습용 웹앱을 활용하고자 하는 방문자

2. 핵심 UI/UX 핵심 가치 (Core UI/UX Values)
앱 런처(App Launcher) 스타일: 운영체제(OS)의 앱 패널이나 스마트폰 홈 화면처럼, 사용자가 앱을 하나의 '독립된 프로그램'으로 인지하고 쉽게 실행할 수 있는 구조 지향

직관적인 탐색: 복잡한 뎁스(Depth)를 배제하고, 메인 화면에서 전체 앱 확인 및 카테고리 필터링이 즉시 가능하도록 설계

웹 기반 관리: 소스코드 수정과 재배포 없이 웹 UI 상에서 직접 데이터를 동적으로 관리 및 변경 가능

3. 주요 기능 및 요구사항 (Key Features & Requirements)
3.1. 메인 레이아웃 및 앱 런처 디스플레이
그리드 뷰 (Grid View): 런처 스타일의 사각형 카드/아이콘 형태로 웹앱들을 배열

앱 카드 구성 요소:

앱 아이콘/썸네일: 등록된 이미지 파일 또는 이미지 URL 링크를 기반으로 출력

앱 명칭: 명확하고 시인성 높은 폰트 적용

카테고리 태그: 해당 앱이 속한 학습 분야 표시

런처 인터랙션: 앱 카드 클릭 시 지정된 실행 링크(외부/내부 URL)로 즉시 새 탭 이동하여 실행

3.2. 카테고리 및 필터링 시스템
상단 고정 카테고리 바 (Category Bar): 화면 상단에 탭 형태로 카테고리를 배치하여 빠른 전환 유도

기본 탭 구성:

전체 (All): 등록된 모든 웹앱을 격자형으로 정렬

관리자가 동적으로 추가한 커스텀 카테고리 노출

동적 필터링: 페이지 새로고침 없이, 탭 클릭 시 해당하는 앱 카드들만 부드러운 애니메이션과 함께 필터링

3.3. 네비게이션 및 편의 기능
검색창 (Search Bar): 앱 명칭이나 태그를 기반으로 한 실시간 타이핑 필터링 기능

테마 토글: 야간 학습자를 위한 라이트 모드 / 다크 모드 전환 지원

4. 관리자 전용 기능 (Admin Functions)
4.1. 관리자 인증 (Authentication)
별도의 회원가입 절차 없이, 사전에 설정된 '관리자 코드(Access Code)'를 입력하여 관리자 권한 획득

인증 성공 시 화면 내 '앱 추가' 및 '카테고리 편집' UI가 활성화됨

4.2. 신규 웹앱 등록 (App Launcher Management)
입력 필드 항목:

제목 (Title): 웹앱의 이름 (필수)

카테고리 선택 (Category Select): 기존 등록된 카테고리 목록 중 하나를 드롭다운으로 선택 (필수)

실행 링크 (URL): 웹앱이 구동되는 실제 웹 주소 (필수)

썸네일 사진 (Thumbnail): 이미지 파일 직접 업로드 또는 외부 이미지 주소(URL) 입력 지원

동작: '등록하기' 버튼 클릭 시 실시간 반영되어 메인 런처에 새로운 카드로 추가됨

4.3. 카테고리 관리 (Category Management)
카테고리 추가: 새로운 학습 카테고리명을 입력하여 실시간 추가

카테고리 수정/삭제: 기존 카테고리의 명칭을 변경하거나 삭제 가능

6. 기술 및 비기능 요구사항 (Technical Requirements)
데이터 영속성 유지 (Storage): 관리자가 웹 상에서 입력 및 편집한 데이터가 휘발되지 않도록 외부 DB(Firebase, Supabase 등) 또는 백엔드 API 시스템과 연동 필수

보안: 관리자 인증 코드는 클라이언트 소스코드에 평문 노출되지 않도록 환경 변수(Environment Variables) 처리 및 서버 측 검증 수행

반응형 웹 디자인 (Responsive Web): 모바일, 태블릿, 데스크톱 등 다양한 디바이스 환경에서 Grid 레이아웃이 유연하게 깨지지 않고 배치되어야 함
